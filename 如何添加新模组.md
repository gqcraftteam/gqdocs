# 如何添加新模组文档 - 快速指南

## 📚 系统说明

您的官网现在已经实现了**独立模组文档系统**。每个模组都有自己的独立HTML文档，便于管理和更新。

### 目录结构
```
mods/
├── mod1.html          ← 示例模组 1
├── mod2.html          ← 示例模组 2  
├── TEMPLATE.html      ← 【参考】模组文档模板
└── 【这里放您新的模组文档】
```

## 🚀 三步快速添加新模组

### 第一步：复制模板文件

1. 在 `mods/` 文件夹中复制 `TEMPLATE.html`
2. 改名为您的模组名称，例如：`my-awesome-mod.html`

### 第二步：编辑模组文档

打开 `my-awesome-mod.html`，修改以下内容：

```html
<!-- 修改标题 -->
<h1>【图标】 您的模组名称</h1>
<p>您的模组简短描述</p>

<!-- 修改版本号 -->
<span style="...">v1.0.0</span>

<!-- 修改各个章节内容 -->
- 模组概述（开发者、版本、MC版本等）
- 功能特性（列出主要功能）
- 安装说明（安装步骤）
- 使用教程（使用方法）
- 配置选项（可选）
- 常见问题（FAQ）
```

### 第三步：在主文档中添加模组卡片

编辑 `docs.html` 文件，在 `.mods-grid` 部分添加：

```html
<div class="mod-card">
    <div class="mod-header">
        <h3>【图标】 您的模组名称</h3>
        <span class="mod-version">v1.0.0</span>
    </div>
    <p class="mod-description">这是模组的描述</p>
    <div class="mod-features">
        <span class="feature-tag">功能1</span>
        <span class="feature-tag">功能2</span>
    </div>
    <a href="mods/my-awesome-mod.html" class="mod-link">查看完整文档 →</a>
</div>
```

## 💡 模组卡片说明

| 部分 | 说明 | 示例 |
|------|------|------|
| 图标 | 使用emoji作为模组图标 | 📦 🔨 🎮 ⚙️ 🔧 |
| 模组名称 | 您的模组名称 | 示例模组 1 |
| 版本号 | 当前版本 | v1.0.0 |
| 描述 | 一句话描述模组功能 | 为游戏添加新的... |
| 功能标签 | 模组的主要功能 | 功能1, 功能2 |
| 文档链接 | 指向模组文档的链接 | mods/your-mod.html |

## 📝 文档内容指南

### 必需内容

**1. 模组概述**
- 模组简介
- 开发者名称
- 当前版本
- 支持的Minecraft版本
- 依赖的Forge版本

**2. 功能特性**
- 列出3-5个主要功能
- 用项目符号或编号

**3. 安装说明**
- 前置条件（MC版本、Forge版本等）
- 逐步安装指引
- 验证安装是否成功

**4. 使用教程**
- 基础用法
- 常见操作步骤
- 可以包含代码示例

**5. 常见问题**
- 用户可能提出的问题
- 清晰的答案

### 可选内容

- **配置选项** - 如果模组可配置
- **API参考** - 如果提供API给其他开发者
- **更新日志** - 版本历史
- **支持反馈** - 联系方式

## 🌐 部署建议

完成文档后，可以部署到：

- **GitHub Pages** - 免费，支持自定义域名
- **Netlify** - 简单快速，自动部署
- **Vercel** - 高性能，支持实时预览
- **您的服务器** - 完全控制

所有这些平台都支持纯静态网页。

## 📞 遇到问题？

1. 查看现有的示例：`mods/mod1.html` 和 `mods/mod2.html`
2. 参考模板：`mods/TEMPLATE.html`
3. 检查 `docs-style.css` 了解样式
4. 确保所有文件路径都相对正确（`../styles.css`）

**提示**：每次添加新模组后，可以在文件夹中右键 → 查看源代码，来快速查看 HTML 结构。
